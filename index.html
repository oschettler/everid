What is this?


It uses the free Github Pages as hosting platform. This means, the resulting site (layout, configuration and content) lives in one of your Git repositories at Github and can be edited to your liking. No lock-in!

EverID currently allows you to choose one of the themes from Bootswatch to style your site. 

How it works

EverID is written as an open source application in PHP. It uses an Sqlite database to make it easy to host the software yourself. 

But you don’t have to. At everid.net you can log-in with your Evernote credentials, using OAuth. This means we don’t get your Evernote username and password, but only an authentication token that you can revoke at any time.

EverID uses authenticated access to the Evernote API to access your notebook and pull those notes from it that have the tag „published“.

Whenever you update your notes or create new ones, EverID gets notified through a web hook. So, once configured, you continue to use Evernote to author pages and have to touch everid.net only if you want to update the site configuration or add a navigation entry.

By the way, to provide configuration, EverID uses Dave Winer’s excellent Concord outliner.

On the Github side, we use OAuth there as well and the Github API to write the pages of your site to a branch gh-pages in the Git repository you have configured. We even have implemented a tiny PHP-based API for this that you could use in your own projects. 

From this repository, Github uses their Jekyll software to create a static website on their Pages facility.
